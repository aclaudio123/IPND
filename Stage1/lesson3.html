Lesson 3: Adding CSS Style to HTML Structure

CSS

CSS stands for Cascading Style Sheets and it gives web developers a way to control the style of related HTML elements. This is done by giving similar HTML elements the same class name and then specifying the style rule that should apply to that class.

Syntactically, in CSS you first write a selector which defines what elements on the page a style rule will apply to then inside the selector you specify the CSS property and value to be applied to that selector. Click here for more expanation of CSS selectors.

Avoiding Repetition

Avoiding repetition is very important in programming because it helps you avoid making errors and also helps with consistency. The reason is when programmers don't avoid repetition, they end up doing the same thing over and over again which becomes time consuming.

In web deveopment, understanding CSS helps with this because CSS style rules can be applied to similar HTML elements grouped together. For example, making a change to an element without dilligently making the same change everywhere that element is used might cause problems. But using CSS, you would make that same change only in the CSS selector used for grouping the HTML elements.

Box Sizing

All HTML elements are boxes, and each box has four components: Margin, Border, Padding and Content. Because there are so many components to each box, it can often be hard to get the size of a box just right. However, there are two techniques you can use to help deal with sizing issues.

  1. Set the size in terms of percentages rather than pixels so that the box size changes depending on how big the screen is.
  2. Set the box-sizing CSS attribute to border-box for every element.

Note: Different browsers work slightly differently and sometimes this causes different browsers to display the same code differently.

Box Positioning

By default, boxes take up the entire width of a page. So in order to posistion boxes, you can use the flexible box attribute flex which provides an efficient way to layout a line and distribute space in a box.

This is done by adding the rule display: flex; to the appropriate CSS which will override the box behavior and let boxes appear next to each other.

Code, Test, Refine

Always remember that building a webpage is an iterative process of coding, testing and refining. Below are a few steps to guide you when building any webpage:

  1. Look for natural boxes. Everything on the web is a box.
  2. Look for repeated styles and semantic elements. Think about how you can relate everything in terms of HTML elements.
  3. Write your HTML.
  4. Apply styles (from Biggest to smallest elements). That is biggest site wide elements to smallest widgets.
  5. Fix Things (and verify). Fix the small things by looking at your site on different browsers and resizing windows.
